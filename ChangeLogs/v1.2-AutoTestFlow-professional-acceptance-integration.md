# v1.2 - AutoTestFlow Professional Acceptance Integration

Date: 2026-06-29

## 中文

### 背景

`Professional_experience` 不应参与 AutoTestFlow 的故障匹配。它应作为专业验收层，用于测试计划、测试用例设计、执行证据、AI/Agent readiness 和测试报告。

### 变更

- 新增 `AutoTestFlow/scripts/professional_acceptance.py`。
- 实现 Phase A：生成 `.state/professional_acceptance.json`，供 stage5 输出专业验收、发布门禁和残余风险矩阵。
- 实现 Phase B：生成 `.state/professional_case_guidance.json`，供 stage3b 添加 `acceptance_refs`、`evidence_required` 和 `professional_notes`。
- 实现 Phase C：生成 `.state/professional_acceptance.seed.json` 与 `.state/professional_acceptance.code_gaps.json`，用于测试计划和代码可测性缺口提示。
- 实现 Phase D：生成 `.state/ai_eval_readiness.json`，用于 AI/Agent eval dataset、grader、redteam、tool-call trace、监控和漂移就绪度。
- 更新 README、SKILL、stage3b 和 stage5 模板。
- 扩展 `validate_knowledge_base.py`，校验可选的 Professional_experience runtime 字段。
- 将 AutoTestFlow 版本更新为 `1.2.0`。

### 运行时原则

- `professional_acceptance.py` 是纯确定性脚本，不启动 LLM。
- Professional_experience 只产生 advisory gate，不生成强断言。
- 所有强断言仍必须来自 `contract.md` 的 `spec-required` oracle。

## English

### Background

`Professional_experience` should not participate in AutoTestFlow fault matching. It should act as a professional acceptance layer for test planning, test case design, execution evidence, AI/Agent readiness, and test reports.

### Changes

- Added `AutoTestFlow/scripts/professional_acceptance.py`.
- Implemented Phase A: generate `.state/professional_acceptance.json` for stage5 professional acceptance, release gates, and residual-risk matrices.
- Implemented Phase B: generate `.state/professional_case_guidance.json` so stage3b can add `acceptance_refs`, `evidence_required`, and `professional_notes`.
- Implemented Phase C: generate `.state/professional_acceptance.seed.json` and `.state/professional_acceptance.code_gaps.json` for test planning and code testability gaps.
- Implemented Phase D: generate `.state/ai_eval_readiness.json` for AI/Agent eval dataset, grader, redteam, tool-call trace, monitoring, and drift readiness.
- Updated README, SKILL, stage3b, and stage5 templates.
- Extended `validate_knowledge_base.py` to validate optional Professional_experience runtime fields.
- Updated AutoTestFlow version to `1.2.0`.

### Runtime Principles

- `professional_acceptance.py` is deterministic and does not start an LLM.
- Professional_experience only produces advisory gates; it must not create strong assertions.
- All strong assertions must still come from `contract.md` `spec-required` oracles.
