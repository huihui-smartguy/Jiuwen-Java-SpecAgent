# v2.8 AutoTestFlow Final Issue And Markdown Companions / AutoTestFlow 最终 Issue 与 Markdown 伴生产物

## 中文

### 背景与根因

- Stage 7 之前会保留 `issue.md`、`root_cause.md`、`fix_solution.md` 等独立文件，但提交到 GitHub Issue 的正文没有稳定合并所有诊断、定位、修复和复验信息，导致最终问题描述容易分散。
- Stage 1 与 Stage 3 的 JSON 产物适合机器处理，但 LLM 和人工复核阅读时需要在多个 JSON 文件之间跳转，场景、分支、用例、oracle、fault 与证据链不够自然。
- Stage 6 模板没有明确要求三个诊断文件必须是可合并的自然语言段落，影响 Stage 7 进行确定性合成。

### 行为变更

- Stage 7 `submit_remediation.py` 会在任何 GitHub 副作用之前生成 `Remediation/issue_bodies/{case}.md`；dry-run 同样生成最终 issue body，但不会调用 `gh`。
- 最终 issue body 合并问题描述、根因、问题定位过程、修复构造、复验证据、结构化证据、补丁摘要与回归摘录，同时保留 Stage 6 的原始分文件用于审计。
- Stage 6 稳定版与 beta 模板要求 `issue.md`、`root_cause.md`、`fix_solution.md` 写成自然语言、信息充分、可确定性合并的材料。
- 新增 `scripts/render_design_markdown.py`，从 JSON 只读渲染 Markdown 伴生产物：
  - `FeatureAnalysis/s1_scenario_examples.md`
  - `FeatureAnalysis/s3a_scenario_landscape.md`
  - `TestCases/test_examples.md`
- Markdown 伴生产物保留 JSON 来源路径与关键追踪字段，包括 `test_suggestion_refs`、`oracle_refs`、`fault_ref`、`fault_oracles`、`acceptance_refs` 与证据要求。

### 兼容性

- Stage 1 / Stage 3 JSON 仍是权威机器可读来源；Markdown 只是稳定、可读、可追踪的投影。
- TestKnowledgeBase registry、contract 与 schema 策略不变，本次不引入知识库 schema 变更。
- Stage 6 原始文件仍保留在归档中，供审计、调试和人工复核使用。

### 验证

- 新增 Stage 7 final issue 组装单元测试，覆盖 `root_cause.md` 与 `fix_solution.md` 合并、dry-run 生成 issue body、无 `gh` 调用。
- 新增 Markdown 渲染单元测试，覆盖 Stage 1 场景卡、Stage 3 场景景观、Stage 3 测试用例中的引用、优先级、预期结果、fault、oracle 与证据字段。
- 目标回归包括 Stage 6 iteration-2、测试建议覆盖、输出布局解析与知识库校验。

## English

### Background And Root Causes

- Stage 7 previously preserved separate `issue.md`, `root_cause.md`, and `fix_solution.md` files, but the GitHub issue body was not guaranteed to merge every diagnosis, positioning, fix, and reverify detail into one coherent final report.
- Stage 1 and Stage 3 JSON artifacts are good for machines, but LLM and human review had to jump across multiple JSON files to understand scenarios, branches, cases, oracles, faults, and evidence.
- Stage 6 templates did not explicitly require the three diagnostic files to be natural-language, merge-ready source material, which made deterministic Stage 7 assembly harder.

### Behavior Changes

- Stage 7 `submit_remediation.py` now writes `Remediation/issue_bodies/{case}.md` before any GitHub side effect. Dry-run mode also creates the final issue body and still makes zero `gh` calls.
- The final issue body merges the problem statement, root cause, problem positioning process, fix construction, empirical reverify evidence, structured evidence, patch summary, and regression excerpt while preserving the original Stage 6 files for audit.
- Stable and beta Stage 6 templates now require `issue.md`, `root_cause.md`, and `fix_solution.md` to be natural-language, sufficiently detailed, and suitable for deterministic merging.
- Added `scripts/render_design_markdown.py` to render Markdown companions from JSON only:
  - `FeatureAnalysis/s1_scenario_examples.md`
  - `FeatureAnalysis/s3a_scenario_landscape.md`
  - `TestCases/test_examples.md`
- Markdown companions preserve JSON provenance and key traceability fields, including `test_suggestion_refs`, `oracle_refs`, `fault_ref`, `fault_oracles`, `acceptance_refs`, and evidence requirements.

### Compatibility

- Stage 1 / Stage 3 JSON remains the authoritative machine-readable source; Markdown is only a stable, readable, traceable projection.
- TestKnowledgeBase registry, contract, and schema policy are unchanged; this release does not introduce a knowledge-base schema change.
- Original Stage 6 files remain in the archive for audit, debugging, and manual review.

### Verification

- Added Stage 7 final-issue assembly tests for merging `root_cause.md` and `fix_solution.md`, generating issue bodies in dry-run, and avoiding `gh` calls.
- Added Markdown renderer tests for Stage 1 scenario cards, Stage 3 scenario landscape, and Stage 3 test examples with refs, priority, expected result, faults, oracles, and evidence.
- Targeted regression coverage includes Stage 6 iteration-2, requirement suggestion coverage, output layout resolution, and knowledge-base validation.
