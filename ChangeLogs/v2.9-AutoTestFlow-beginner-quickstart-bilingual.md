# v2.9 AutoTestFlow Beginner Quickstart / AutoTestFlow 入门 Quickstart

## 中文

### 背景

默认全链路命令使用 `autotestflow.suts.md` 与 `remediation.config.json`，但新用户不一定能立刻找到对应 schema、模板和校验方式。此次调整把入门准备直接放到默认入口，避免用户先看到命令、后找输入文件。

### 主要变化

- 新增 `AutoTestFlow/examples/quickstart/`，提供默认文件名的 starter：
  - `autotestflow.suts.md`
  - `remediation.config.json`
  - `README.md`
- `AutoTestFlow/README.md` 的 `默认全链路命令（推荐）` 小节内嵌 Quickstart，先说明复制 starter、修改输入、运行校验，再展示原默认命令。
- `AutoTestFlow/SKILL.md` 的默认命令说明补充缺失文件处理：找不到 `autotestflow.suts.md` 或 `remediation.config.json` 时，优先指向 quickstart、完整示例与 schema。
- `AutoTestFlow/reference/remediation_config.py` 新增只读校验 CLI：
  `python AutoTestFlow/reference/remediation_config.py --check remediation.config.json`。

### 兼容性与安全

- 默认 `/auto-test-flow` 命令不变。
- Quickstart remediation 配置默认 `switches.allow_open_issue=false`，不会外发 evidence issue。
- 缺少或非法 remediation 配置时仍跳过 stage6/7 并说明原因，不强行修复。

### 验证

- Quickstart SUT 描述通过 `sut_manifest.py` 解析与归一化。
- Quickstart remediation 配置通过新的 `--check` CLI。
- 文档守卫覆盖 README/SKILL 默认入口中的 quickstart、schema 与默认文件名引用。

## English

### Background

The default full-pipeline command depends on `autotestflow.suts.md` and `remediation.config.json`, but new users may not immediately find the matching schemas, templates, or validation commands. This update moves the setup path into the default entry section.

### Key Changes

- Added `AutoTestFlow/examples/quickstart/` with starter files using the default filenames:
  - `autotestflow.suts.md`
  - `remediation.config.json`
  - `README.md`
- Embedded a Quickstart in the `AutoTestFlow/README.md` default command section: copy starters, edit inputs, validate them, then run the unchanged default command.
- Updated `AutoTestFlow/SKILL.md` default-command guidance so missing `autotestflow.suts.md` or `remediation.config.json` points to quickstart files, full examples, and schemas.
- Added a read-only remediation config validation CLI:
  `python AutoTestFlow/reference/remediation_config.py --check remediation.config.json`.

### Compatibility And Safety

- The default `/auto-test-flow` command is unchanged.
- The quickstart remediation config keeps `switches.allow_open_issue=false` by default, so it will not publish evidence issues.
- Missing or invalid remediation config still skips stage6/7 with a clear reason instead of forcing repair.

### Verification

- The quickstart SUT description parses and normalizes through `sut_manifest.py`.
- The quickstart remediation config passes the new `--check` CLI.
- Documentation guard tests cover quickstart, schema, and default filename references in the README/SKILL default entry.
