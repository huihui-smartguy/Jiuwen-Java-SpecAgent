# 修订记录 v1.0 —— AutoTestFlow / TestKnowledgeBase 主干化与 Specification_Repository 退场

| 项 | 值 |
|---|---|
| 版本 | **AutoTestFlow v1.0.0 / TestKnowledgeBase v1.1.0** |
| 日期 | 2026-06-29 |
| 分支 | `release/autotestflow-v1.0` |
| 范围 | AutoTestFlow 默认知识源、TestKnowledgeBase 治理文件、确定性知识库校验 |
| 兼容性 | 向后兼容：`Specification_Repository` 保留为缺省探测失败后的遗留 fallback，但不再作为后续迭代目标 |

## 一、背景

AutoTestFlow 后续迭代需要一个可持续演进的知识库主干。`Specification_Repository` 已经在早期版本承担故障库和 wiki 派生职责，但它的定位更接近初始规格/故障仓。随着 `TestKnowledgeBase/Fault/` 和 `TestKnowledgeBase/Professional_experience/` 建立，继续让两个目录并行承担实际产品职责会造成路径漂移、版本漂移和晋升口径不一致。

本轮明确产品方向：

> `Specification_Repository` 将被 `TestKnowledgeBase` 替换，后续迭代不再发挥实际产品作用；仅保留为历史示例、旧文档和显式旧调用的兼容路径。

## 二、主要改动

| 区域 | 改动 |
|---|---|
| `AutoTestFlow/scripts/match_faults.py` | 默认优先探测 `TestKnowledgeBase/Fault/rest_api_faults.json`；旧 `Specification_Repository/rest_api_common_faults.json` 仅 fallback。 |
| `AutoTestFlow/scripts/record_faults.py` | 自积累默认 overlay 改为 `TestKnowledgeBase/Fault/project_faults.json`。 |
| `AutoTestFlow/beta/scripts/gen_wiki.py` | Beta wiki 默认从 `TestKnowledgeBase/Fault/rest_api_faults.json` 派生。 |
| `AutoTestFlow/README.md` / `SKILL.md` | 文档声明 TestKnowledgeBase 为后续知识库主干。 |
| `AutoTestFlow/manifest.json` / `VERSION` | 新增 AutoTestFlow 版本与默认知识源声明。 |
| `TestKnowledgeBase/manifest.json` / `VERSION` | 新增知识库版本、替代关系与 source-of-truth 策略。 |
| `TestKnowledgeBase/TRANSFORMATION_SCOPE.md` | 新增迁移范围、运行时规则和验证门禁。 |
| `AutoTestFlow/scripts/validate_knowledge_base.py` | 新增确定性校验脚本，覆盖 JSON 解析、fault_id 唯一性、必需字段和 `source_refs` 完整性。 |

## 三、运行时策略

默认发现顺序：

```text
1. TestKnowledgeBase/Fault/rest_api_faults.json
2. Specification_Repository/rest_api_common_faults.json   # legacy fallback only
3. .claude/skills/AutoTestFlow/shared/fault_library/rest_api_common_faults.json
```

强约束不变：

1. `contract.md` 仍是唯一强断言 Oracle。
2. `TestKnowledgeBase/Fault/` 只决定“哪些故障要测”。
3. `Professional_experience/` 只决定“如何验收、如何报告、何时人工裁决”，不得直接升级断言。
4. `Specification_Repository` 不再接收新的产品迭代能力。

## 四、验证

新增验证命令：

```bash
python AutoTestFlow/scripts/validate_knowledge_base.py
python -m py_compile AutoTestFlow/scripts/match_faults.py AutoTestFlow/scripts/record_faults.py AutoTestFlow/beta/scripts/gen_wiki.py AutoTestFlow/beta/scripts/check_wiki.py AutoTestFlow/scripts/validate_knowledge_base.py
```

故障匹配 smoke 建议使用临时输出目录，显式指定新默认库：

```bash
python AutoTestFlow/scripts/match_faults.py \
  --output-dir <output_dir> \
  --fault-lib TestKnowledgeBase/Fault/rest_api_faults.json
```
