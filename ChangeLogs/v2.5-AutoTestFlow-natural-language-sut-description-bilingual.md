# v2.5 AutoTestFlow Natural-Language SUT Description / 自然语言 SUT 描述

## 中文

### 背景

原 `autotestflow.suts.md` 需要用户按 `autotestflow.sut-manifest.v1`
YAML front matter 填写 `suite`、`targets`、`source`、`runtime`、
`readiness_probe` 等字段。该格式利于机器校验，但对普通使用者不够友好。

### 主要变化

- `--sut-manifest autotestflow.suts.md` 现在优先支持自然语言 Markdown。
- 保留 legacy Markdown+YAML manifest 完全兼容；既有文件可继续使用。
- Stage 0 会把自然语言 SUT 描述转换为 canonical manifest，再由
  `scripts/sut_manifest.py` 确定性校验并写 `.state/sut_manifest.normalized.json`。
- 新增 `.state/sut_description.parse.json` 与
  `.state/sut_description.review.md`，用于展示解析结果、置信度、风险推断与人工确认原因。
- 支持 direct/predeployed target 无源码：`source.available=false` 时跳过源码扫描/code-only GAP，
  但仍可在 SUT 可达时执行契约探测与黑盒测试。
- 推断出 managed build/start 命令、低置信或缺字段时标记 review required；命令执行仍受
  `sut_runtime.py --allow-commands` 二次门控保护。
- 环境变量会保留名称与运行时占位，secret/token/password/key 等敏感值在解析与 review 产物中脱敏。

### 验证

- 扩展 `test_multi_sut_manifest.py` 覆盖：
  自然语言 direct target、managed target、multi-SUT dependency、缺少 base URL、secret 脱敏、
  source-less direct target 跳过代码扫描，以及 parse/review artifact 写入。
- 旧 YAML manifest、legacy `--sut-base-url`、probe/runtime 测试保持通过。

## English

### Background

The previous `autotestflow.suts.md` required users to fill the
`autotestflow.sut-manifest.v1` YAML front matter with fields such as `suite`,
`targets`, `source`, `runtime`, and `readiness_probe`. That shape was
machine-friendly but difficult for users to author correctly.

### Key Changes

- `--sut-manifest autotestflow.suts.md` now prefers natural-language Markdown.
- Legacy Markdown+YAML manifests remain fully compatible.
- Stage 0 converts the natural-language SUT description into the canonical
  manifest shape, then `scripts/sut_manifest.py` validates it deterministically
  and writes `.state/sut_manifest.normalized.json`.
- Added `.state/sut_description.parse.json` and
  `.state/sut_description.review.md` for parse results, confidence, risky
  inferences, and review reasons.
- Direct/predeployed targets can omit source code with `source.available=false`;
  source scan and code-only GAP are skipped, while live contract probing and
  black-box tests still run when the SUT is reachable.
- Managed build/start inference, low confidence, or missing fields are marked as
  review required. Command execution remains protected by the existing
  `sut_runtime.py --allow-commands` gate.
- Environment variable names and runtime placeholders are preserved, while
  secret/token/password/key-like values are redacted in parse and review
  artifacts.

### Verification

- Extended `test_multi_sut_manifest.py` for natural-language direct targets,
  managed targets, multi-SUT dependency prose, missing base URL handling, secret
  redaction, source-less direct target code-scan skipping, and parse/review
  artifact writing.
- Existing YAML manifest, legacy `--sut-base-url`, probe, and runtime tests
  remain covered.
