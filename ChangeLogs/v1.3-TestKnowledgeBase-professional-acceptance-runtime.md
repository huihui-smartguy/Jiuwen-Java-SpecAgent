Date: 2026-06-29

## 中文

### 背景

`Professional_experience` 不应作为 AutoTestFlow 的故障匹配库使用。它的职责是提供测试计划、测试用例设计、AI/Agent readiness、发布门禁和测试报告质量的专业验收知识。

### 变更

- 将 TestKnowledgeBase 版本更新为 `1.3.0`。
- 为 `Professional_experience/acceptance_criteria.json` 增加 `professional_acceptance.v1` runtime 字段：
  - `applies_to`
  - `stage_scope`
  - `auto_evaluable`
  - `evidence_map`
  - `default_status_when_missing`
  - `risk_scope`
- 更新 `TestKnowledgeBase/registry.json` 中的 `professional.acceptance` package，明确 Phase A-D 集成产物。
- 新增 `Professional_experience/runtime_schema.md`，定义专业验收 runtime artifact 合约。
- 更新 `Professional_experience/guide.md` 与 `autotestflow_mapping.md`，说明 Phase A-D 的消费路径。

### 运行时原则

- `Professional_experience` 只能产生 advisory gate，不生成强断言。
- 所有 L2 断言仍必须来自 `contract.md`。
- AI/Agent 标准在非 AI 项目中应判为 `not_applicable`，避免误报。

## English

### Background

`Professional_experience` should not be used as AutoTestFlow's fault-matching library. Its role is to provide professional acceptance knowledge for test planning, test case design, AI/Agent readiness, release gates, and report quality.

### Changes

- Updated TestKnowledgeBase version to `1.3.0`.
- Added `professional_acceptance.v1` runtime fields to `Professional_experience/acceptance_criteria.json`:
  - `applies_to`
  - `stage_scope`
  - `auto_evaluable`
  - `evidence_map`
  - `default_status_when_missing`
  - `risk_scope`
- Updated the `professional.acceptance` package in `TestKnowledgeBase/registry.json` with Phase A-D integration artifacts.
- Added `Professional_experience/runtime_schema.md` to define the professional acceptance runtime contract.
- Updated `Professional_experience/guide.md` and `autotestflow_mapping.md` with the Phase A-D consumption path.

### Runtime Principles

- `Professional_experience` may only produce advisory gates; it must not create strong assertions.
- All L2 assertions must still come from `contract.md`.
- AI/Agent criteria should become `not_applicable` for non-AI projects to avoid false warnings.
