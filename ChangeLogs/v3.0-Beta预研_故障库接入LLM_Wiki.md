## 一、背景与动机

- **问题**：`07` 评估"故障库能否以 LLM Wiki 形式供 AutoTestFlow 使用"，结论为**可以，但仅作 advisory NL 层**——
  适配 LLM 阶段（stage2.6b / stage6），不适配确定性脚本（`match_faults.py`/`record_faults.py`），且**不作 oracle、不替代 matcher 结构化输入**。
- **缺口**：该结论尚停留在分析文档，未有可运行的工程预研；且新特性属"预研"，直接并入稳定流水线有破坏内核风险。
- **方案**：新增 **Beta 预研区**（物理隔离 + 开关门控），落地 `07` §九 **Phase A**——由结构化故障库**单向派生**仓内 NL 文章，
  供 stage2.6b / stage6 的 LLM 子 Agent 按 `fault_id` 确定性取用，作 advisory 建议层。
- **核心决策（锁定）**：① **开关 + `beta/` 隔离**（非独立副本），复用 v2.0 `--remediate` 的"默认 off + 字节级一致"范式；
  ② **单一真相在 JSON**、wiki 单向派生；③ wiki **不作 oracle、不进 matcher、断言由 `contract.md` 封顶**；
  ④ 全程**离线确定性**可验证，**不接入真实 SUT/LLM**（逻辑正确即可）。

## 二、总体架构（数据流）

```
Specification_Repository/rest_api_common_faults.json   ← 单一真相（source of truth）
   │
   ├──► scripts/match_faults.py / record_faults.py      [确定性消费者：只吃 JSON；★本轮零改动]
   │
   └──(beta/scripts/gen_wiki.py 单向派生; 仅 --beta-wiki on)
         └──► Specification_Repository/wiki/{fault_id}.md (+category/index)   [advisory NL 层]
                │  每篇溯源 fault_id(+category)；不声明无条件断言级别
                ├──► stage2.6b(beta 模板)  读 wiki/{fault_id}.md 辅助绑定 validation_point→specId
                └──► stage6  (beta 模板)  读 wiki/{fault_ref}.md 丰富 issue「规格库知识」半边
                       （判据仍由 contract.md 封顶；wiki 不作 oracle）
                              ▲
         beta/scripts/check_wiki.py 把护栏（覆盖/溯源/不漂移/不越权/非matcher）变为可执行断言（离线机械校验）
```

四条贯穿原则（承接 v1/v2 内核）：**契约唯一权威**（断言由 `contract.md` 封顶）、**不自我认证**（wiki 不作判据）、
**文件即协议 / 优雅降级**（wiki 是仓内文件；off 即旧行为字节级一致）、**单一真相单向派生**（JSON→wiki，绝不反写）。

## 三、分阶段改动

### Beta 区脚手架（物理隔离）

- 新增 `AutoTestFlow/beta/` 子树：`scripts/`（生成器 + 校验器）、`templates/`（两 beta 模板）、`shared/`（纪律 + schema）、`examples/`（离线 demo）、`README.md`。稳定目录（`scripts/`、`templates/`、`reference/`、`shared/`）**零改动**。

### gen_wiki.py —— 故障库 → wiki 单向派生（纯脚本，确定性）

- 读 `rest_api_common_faults.json`(+overlay)，复用 `match_faults.py` 的 `load_faults`/`_norm_cat`/`_substitute` 同构逻辑（自包含，不 import 稳定脚本）。
- 为每个 `fault_id` 产一篇文章（溯源头部 / 通俗描述 / 触发 / 预期+验证点 / 候选 specId 种类+封顶免责 / 按类通用根因 advisory / 历史案例 / 关联故障），外加每类一篇汇总 + 一篇索引。
- **幂等**（连跑字节一致）、**缺库/`--beta-wiki off` 优雅退出**。

### check_wiki.py —— 护栏校验器（纯脚本，离线）

- C1 覆盖（fault_id↔文章一一对应）/ C2 溯源（头部回链）/ C3 不漂移（重渲染逐字节比对，证单向派生）/ C4 不越权（advisory 免责齐备、无判据化措辞）/ C5 非 matcher 输入（`match_faults`/`record_faults` 不引用 wiki）。
- 产 `.state/wiki_check.json`；全绿退 0，任一不过退非零。

### stage2.6b / stage6 的 beta 模板（= 稳定模板 + 一步 advisory wiki）

- `beta/templates/stage2_6_fault_match.beta.md`：`enrich.needs_bind=true` 时读 `wiki/{fault_id}.md` 辅助把 unbound 验证点绑定到 specId；**断言仍按 contract 封顶**，wiki 不命中 contract 则不绑定。
- `beta/templates/stage6_defect_analyze.beta.md`：`fault_ref` 命中时读 `wiki/{fault_ref}.md` 作根因方向 + issue 叙事素材；**判据仍唯一来自 contract**，根因须由真实源码+trace 坐实。
- 两模板逐字保留稳定模板全部红线；wiki 缺失即跳过该步、行为与稳定模板一致。

### 编排器接入（SKILL.md，仅 additive）

- 参数表新增 `--beta-wiki off|on`；stage2.6 步骤3 / stage6 步骤2 各加一条 `--beta-wiki on` 分支注；新增「Beta（预研特性，默认关闭）」节。`off`（默认）时不触发任何 beta 路径。

## 五、新增数据契约（schema）

| 契约/文件 | 说明 |
|---|---|
| `Specification_Repository/wiki/{fault_id}.md` | 单故障 NL 文章；固定章节顺序；头部强制溯源 + advisory 免责。格式见 `beta/shared/wiki_schema.md` |

| `.state/wiki_check.json` | 校验报告（`ok` / `checks[C1..C5]` / `summary`） |
| `--beta-wiki` | Beta 总门控：`off`（默认，字节级一致）/ `on`（生成+使用 wiki） |

## 六、行为与兼容性

- **优雅降级（硬保证）**：`--beta-wiki=off`（默认）→ 不跑 gen_wiki、不读 wiki、用稳定模板，流水线与 v2.x **字节级一致**。
- **单一真相单向派生**：JSON 是 source of truth；wiki 由 `gen_wiki.py` 单向编译，禁止手改（`check_wiki.py` C3 检出漂移）。
- **不自我认证 / 不作 oracle**：判据唯一来自 `contract.md`；wiki 仅 advisory，与 contract 冲突一律以 contract 为准。
- **不污染确定性路径**：`match_faults.py`/`record_faults.py` 零改动、永不读 wiki（C5 守卫）。
- **wiki 缺失安全**：某 fault 无文章 / `fault_ref` 为空 → beta 模板跳过 wiki 步，逻辑与稳定模板一致。

## 九、使用指引

```bash
# 启用 Beta LLM Wiki（先派生 + 校验，再在流水线用 beta 模板）
python AutoTestFlow/beta/scripts/gen_wiki.py            # 故障库 JSON → Specification_Repository/wiki/*.md（单向派生、幂等）
python AutoTestFlow/beta/scripts/check_wiki.py          # 护栏校验（全绿才用）→ .state/wiki_check.json
/auto-test-flow 需求.md <java仓>/<模块> --sut-base-url http://host:port \
    --faults on --fault-enrich on --beta-wiki on

# 关闭（默认，与未接入 Beta 一致）
... --beta-wiki off

# 复现离线 demo（无需 SUT/LLM）
DEMO=AutoTestFlow/beta/examples/a2a/wiki_demo
python AutoTestFlow/beta/scripts/gen_wiki.py   --fault-lib $DEMO/faults.sample.json --wiki-dir /tmp/wd/wiki
python AutoTestFlow/beta/scripts/check_wiki.py --fault-lib $DEMO/faults.sample.json --wiki-dir /tmp/wd/wiki --output-dir /tmp/wd
diff -r $DEMO/golden/wiki /tmp/wd/wiki && echo "WIKI MATCHES GOLDEN"
```

- 关闭：`--beta-wiki off`（默认）。
- 改知识：改 `rest_api_common_faults.json`(+overlay) → 重跑 `gen_wiki.py` → `check_wiki.py` 复验（**勿手改 wiki/**）。
- 详见 `AutoTestFlow/beta/README.md`、`beta/shared/wiki_rules.md` 与 `beta/examples/a2a/wiki_demo/README.md`。
