# v1.1 - AutoTestFlow Generalized Knowledge Matching

Date: 2026-06-29

## Background

AutoTestFlow v1.0 preferred `TestKnowledgeBase/Fault/rest_api_faults.json`, but the actual matcher and recorder still followed the legacy `Specification_Repository` data shape and REST/API-specific category heuristics. That kept Web, Agent, DFX, and future TestKnowledgeBase packages from becoming first-class runtime inputs.

## Changes

- Added `AutoTestFlow/scripts/knowledge_base.py` as a deterministic TestKnowledgeBase adapter.
- Refactored `match_faults.py` to discover TestKnowledgeBase packages through `registry.json` or `Fault/*.json` glob fallback.
- Added `--knowledge-root` and `--knowledge-domain` to support package/domain filtering.
- Stopped automatic fallback to `Specification_Repository`; legacy files now require explicit `--fault-lib`.
- Added `.state/knowledge_matches.json` as the primary stage2.6 output while preserving `.state/fault_matches.json` for stage3b/stage5 compatibility.
- Refactored matching to use package/domain/category metadata first, with legacy tag/category heuristics as fallback.
- Refactored `record_faults.py` to emit `.state/new_knowledge_candidates.json` and retain `.state/new_faults_detected.json` as compatibility output.
- Extended `validate_knowledge_base.py` to validate TestKnowledgeBase registry declarations when present.
- Updated AutoTestFlow version to `1.1.0` and refreshed README/SKILL/templates.

## Compatibility

- Existing stage3b and stage5 flows continue to read `.state/fault_matches.json`.
- Existing `fault_ref` fields remain valid and may point to either `fault_id` or `knowledge_id`.
- Old Specification_Repository demos remain possible only through an explicit `--fault-lib Specification_Repository/...` argument.

## Validation Focus

- Default TestKnowledgeBase discovery produces a deterministic match plan.
- `--faults off` still exits without producing match files.
- `record_faults.py` only records `sdk_defect` candidates and does not collect `sut_unsatisfied`, `harness_defect`, or `env_issue`.
- Knowledge-base validation remains offline and deterministic.
