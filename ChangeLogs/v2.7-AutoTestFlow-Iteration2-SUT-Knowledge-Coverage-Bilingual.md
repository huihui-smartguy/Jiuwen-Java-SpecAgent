# v2.7 AutoTestFlow Iteration 2 - SUT / Knowledge / Coverage

## 中文

### 背景与根因

- SUT 自然语言解析过早把所有 `[section]` 当作目标，导致 `[LLM]`、`[source_path]` 等全局配置被误识别；`name = EDPA` 又被当成环境变量，真实目标缺少源码信息后容易得到 `source.available=false` 并跳过 Stage 2。
- 故障匹配默认参数本身不是主要问题；实际问题是无源码事实时场景知识域推断偏弱，并且所有质量类故障共享同一截断桶，REST / Agent / Web / DFX 会互相挤占。
- Stage 1 只有通用覆盖规则，没有把需求文档中的“测试建议 / 推荐测试点 / 自测门禁”建立为一等可追踪对象，无法证明建议已映射到场景、分支或用例。

### 行为变更

- SUT 解析新增 section 分类：target / source / config / notes。`[LLM]` 写入 suite 级红acted环境配置，`[source_path]` 写入全局源码元数据，不再生成假目标。
- 支持 `[section] inline text`、`key = value`、`environmentN` 泛化段落。示例中的目标会规范化为 `edpa`、`versatile_mock`、`front_tool`。
- 全局 `source_path` 只应用到主目标；遮蔽路径记录为 `source.redacted=true` 与人工复核原因，远程 GitHub 地址记录为 `source.remote_url`，本迭代不自动 clone。
- 多目标场景中的角色推断会标记 `role_inference_requires_review`，便于人工确认拓扑。
- `match_faults.py` 只有显式传入 `--knowledge-domain` 时才过滤知识包；省略参数时加载全部 registry 包。
- 场景知识域推断新增目标元数据、源码/profile线索、服务名（如 `front_tool`）与场景文本综合判断；DFX 作为跨域知识保持可匹配。
- 故障截断从单一分支桶改为按 domain + branch 独立配额，并在 `knowledge_matches.meta.stats.domain_breakdown` 输出 considered / matched / filtered / truncated 明细。
- Stage 1 新增 `s1_index.test_suggestions[]`，场景、分支、Stage 3b 用例支持 `test_suggestion_refs`；merge 阶段输出测试建议覆盖报告，未引用且未标记不适用时触发覆盖错误。

### 兼容性

- 保留既有字段：`source.path`、`source.available`、`knowledge_matches`、`fault_matches`、`fault_ref`、`fault_oracles`。
- 新增字段为可选扩展：`source.remote_url`、`source.redacted`、`test_suggestion_refs`、`test_suggestions`、`domain_breakdown`。

### 验证

- 新增 EDPA 多 SUT 自然语言解析回归：验证 3 个真实目标、无 `llm` 假目标、主目标源码映射、LLM 配置脱敏、角色推断复核。
- 新增源码可用性回归：遮蔽路径不可用，真实本地路径才允许源码扫描。
- 新增 Agent / Web / DFX 故障匹配回归：默认加载全部知识域，per-domain 配额保留非 REST 故障。
- 新增测试建议覆盖回归：每条建议必须被用例引用或显式标记 `not_applicable`。

## English

### Background And Root Causes

- Natural-language SUT parsing treated every `[section]` as a target too eagerly, so `[LLM]` and `[source_path]` could become false targets; `name = EDPA` was parsed as an environment variable, leaving real service targets without source metadata and making Stage 2 easy to skip.
- The fault-matching default was not the main cause. The larger issue was weak scene-domain inference without Stage 2 facts, plus one shared quality truncation bucket where REST / Agent / Web / DFX candidates could starve each other.
- Stage 1 had generic coverage rules but no first-class ledger for explicit “testing recommendations / suggested test points / self-test gates,” so the pipeline could not prove recommendation-to-scenario/case coverage.

### Behavior Changes

- SUT parsing now classifies sections as target / source / config / notes. `[LLM]` becomes suite-level redacted environment metadata, and `[source_path]` becomes global source metadata instead of a target.
- `[section] inline text`, `key = value`, and generic `environmentN` sections are supported. The provided sample normalizes to `edpa`, `versatile_mock`, and `front_tool`.
- A global source path applies only to the primary target. Masked paths are recorded as `source.redacted=true` with review reasons, and fallback GitHub URLs are preserved as `source.remote_url`; this iteration does not auto-clone repositories.
- Multi-target role inference is marked with `role_inference_requires_review` so topology remains reviewable.
- `match_faults.py` filters knowledge packages only when `--knowledge-domain` is explicitly supplied; omitted means load all registry packages.
- Scene-domain inference now uses target metadata, source/profile hints, service names such as `front_tool`, and scenario text. DFX remains eligible as cross-cutting knowledge.
- Fault truncation is now per domain + branch instead of one shared branch bucket. `knowledge_matches.meta.stats.domain_breakdown` reports considered / matched / filtered / truncated counts.
- Stage 1 now emits `s1_index.test_suggestions[]`; scenarios, branches, and Stage 3b cases can carry `test_suggestion_refs`. The merge step reports suggestion coverage and fails when an actionable suggestion is neither referenced nor marked not applicable.

### Compatibility

- Existing fields remain compatible: `source.path`, `source.available`, `knowledge_matches`, `fault_matches`, `fault_ref`, and `fault_oracles`.
- New fields are optional extensions: `source.remote_url`, `source.redacted`, `test_suggestion_refs`, `test_suggestions`, and `domain_breakdown`.

### Verification

- Added EDPA multi-SUT natural-language regression coverage for three real targets, no fake `llm` target, primary-source mapping, redacted LLM config, and role-inference review.
- Added source-availability regression coverage: masked paths are unavailable; real local paths allow source scanning.
- Added Agent / Web / DFX fault-matching regressions: default all-domain loading and per-domain quotas preserve non-REST matches.
- Added recommendation-coverage regression: every suggestion must be referenced by cases or explicitly marked `not_applicable`.
