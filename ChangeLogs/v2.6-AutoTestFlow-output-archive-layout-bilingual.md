# v2.6 AutoTestFlow Output Archive Layout Refactor / AutoTestFlow 输出归档结构重构

## 中文

### 变更摘要

- 在 `feature/autotestflow/iteration-2` 合并最新 `develop` 后完成输出归档结构重构。
- 新增 `AutoTestFlow/scripts/output_layout.py`，统一管理 root 与 target 的规范输出路径，并提供只读 legacy fallback，确保旧归档仍可被脚本读取。
- 新运行不再写散落在 target 根目录或 `.state/` 下的主要产物，改按功能目录归档：
  - `RunMetadata/`
  - `FeatureAnalysis/`
  - `Contract/`
  - `KnowledgeBase/`
  - `QualityGates/`
  - `TestCases/`
  - `TestRun/`
  - `Reports/`
  - `FaultAnalysis/`
  - `Remediation/`
- 保持 `--output-dir` 用户接口不变；normalized manifest 新增 `artifact_dirs` 与 `artifacts`，保留 `contract_path`、`case_results_path`、`report_path` 等既有字段，但指向规范路径。
- 更新阶段模板、AutoTestFlow 文档、SUT manifest schema、A2A 示例归档与 TestKnowledgeBase Professional_experience artifact 引用。

### 兼容性说明

- 新输出只写规范目录，不镜像 legacy flat 文件。
- 旧归档仍可通过脚本 fallback 读取，例如旧的 `contract.md`、`test_design.json`、`case_results.json`、`.state/results/*.json`、`.state/fault_matches.json` 等。
- TestKnowledgeBase Professional_experience evaluator 同时接受旧短名与新规范 artifact 名。

### 验证范围

- 新增 output layout resolver 单元测试。
- 更新 multi-SUT manifest、stage2 code scan adapter、fault oracle、stage6 iteration-2 测试夹具。
- 迁移 A2A fault/remediation demo fixture 到规范目录，用于 smoke 验证。

## English

### Summary

- Merged current `develop` into `feature/autotestflow/iteration-2` and refactored AutoTestFlow output archive layout.
- Added `AutoTestFlow/scripts/output_layout.py` as the shared canonical path resolver with read-only legacy fallback support.
- New runs no longer scatter primary artifacts across target root or `.state/`; artifacts are grouped by function:
  - `RunMetadata/`
  - `FeatureAnalysis/`
  - `Contract/`
  - `KnowledgeBase/`
  - `QualityGates/`
  - `TestCases/`
  - `TestRun/`
  - `Reports/`
  - `FaultAnalysis/`
  - `Remediation/`
- Kept the existing `--output-dir` contract. The normalized manifest now includes `artifact_dirs` and `artifacts`, while existing fields such as `contract_path`, `case_results_path`, and `report_path` remain available and now point to canonical locations.
- Updated stage templates, AutoTestFlow docs, SUT manifest schema, A2A demo archives, and TestKnowledgeBase Professional_experience artifact references.

### Compatibility

- New output writes canonical paths only; no legacy mirror files are emitted.
- Existing archives remain readable through fallback resolution for old paths such as `contract.md`, `test_design.json`, `case_results.json`, `.state/results/*.json`, and `.state/fault_matches.json`.
- The Professional_experience evaluator accepts both legacy short artifact names and canonical artifact paths.

### Verification Scope

- Added unit coverage for the output layout resolver.
- Updated multi-SUT manifest, stage2 code scan adapter, fault oracle, and stage6 iteration-2 fixtures.
- Migrated A2A fault/remediation demo fixtures to the canonical layout for smoke validation.
